## 1. システム概要

### 目的

製造現場の測定データから工程能力指数（Cp/Cpk）を自動計算し、工程ごとの「ばらつき」と「中心ずれ」を即座に可視化する。非統計者でも「何が問題か」「どう対処するか」を理解できる改善アクション付きのレポートを生成する。

### C-61（品質管理ポータル）との関係

- C-62 は**独立システム**として実装する（C-61 の拡張ではない）
- C-61 の portal.py からは将来的に軽量ウィジェット経由でリンクできる設計とするが、本仕様には含めない
- 起動コマンド: `streamlit run 02_manufacturing/08_cp_cpk/app.py`

## 4. 計算ロジック（analyze.py）

### Cp / Cpk の定義

```
σ = 標準偏差（ddof=1）
μ = 平均値

Cp  = (USL - LSL) / (6 × σ)           # 規格幅に対するばらつきの比
Cpk = min((USL - μ) / (3σ), (μ - LSL) / (3σ))  # 中心ずれを考慮した能力
```

### 判定基準（Cpk 優先）

| Cpk 値 | 判定 | 色 |
|--------|------|-----|
| ≥ 1.67 | 非常に良好 | #16a34a (green) |
| 1.33 ≤ Cpk < 1.67 | 良好 | #16a34a (green) |
| 1.00 ≤ Cpk < 1.33 | 要改善 | #d97706 (amber) |
| < 1.00 | 不可 | #dc2626 (red) |

### 診断ロジック（改善アクション自動生成）

```python
if Cp >= 1.33 and Cpk < 1.00:
    # 分布幅は十分 → 中心ずれが問題
    action = "平均値を規格中心に調整（センタリング優先）"
    deviation_mm = abs(μ - (USL + LSL) / 2)

elif Cp < 1.00:
    # ばらつきが大きすぎる
    action = "工程のばらつきを低減（設備精度・材料・作業手順の見直し）"

elif 1.00 <= Cp < 1.33:
    # 両方改善余地あり
    action = "ばらつき低減と中心合わせの両立が必要"

else:
    action = "現状維持（定期モニタリング継続）"
```

### 出力データ構造（工程ごと）

```python
{
  "process": "溶接",
  "n": 200,           # サンプル数
  "mean": 10.02,
  "std": 0.087,
  "cp": 1.45,
  "cpk": 0.98,
  "usl": 10.20,
  "lsl": 9.80,
  "verdict": "要改善",   # 良好 / 要改善 / 不可
  "action": "平均値を規格中心に調整（センタリング優先）",
  "out_of_spec_pct": 2.3,  # 規格外推定割合（%）
  "center_deviation": 0.12  # 規格中心からのずれ
}
```

## 5. 画面設計

### 5-1. 全体レイアウト

左右分割（`st.columns([1, 2])`）

```
┌──────────────────┬──────────────────────────────────┐
│  ⚙ 設定パネル    │  📊 結果ダッシュボード              │

│                  │                                   │
│  CSVアップロード  │  工程能力サマリー表                │

│  ▶ 分析実行      │    Cp/Cpk スコアカード             │
```

### 5-2. 設定パネル（左）

**① CSV アップロード**
- `st.file_uploader` + サンプルCSVダウンロードボタン
- アップロード後、工程名の一意リストを自動検出

**② 列マッピング**
- 工程名列 / 測定値列 をドロップダウンで選択
- 日付列はオプション

**③ 規格値入力（アコーディオン型）**
- 工程ごとに `st.expander` で展開
- ヘッダーに設定済（✓緑）/ 未設定（✗赤）バッジ
- 「全工程に同じ値を適用」ボタン → 全 expander に一括コピー
- 入力値は `st.session_state.spec_values[process]` に保持

**④ 分析実行ボタン**
- 未設定工程があれば警告を表示して実行は許可（その工程をスキップ）
- `st.button("▶ 分析実行", type="primary")`

### 5-3. 結果パネル（右）

**① 工程能力サマリー表**

| 工程 | Cp | Cpk | 判定 |
|------|----|-----|------|
| 溶接 | 1.45 | 1.38 | ✓良好 |
| 塗装 | 1.12 | 0.98 | ✗不可 |

- 判定セルを色分け（背景色 + テキスト色）
- 行選択 → `st.session_state.selected_process` を更新 → 詳細パネル切り替え
- 実装: `st.radio` で工程名を選択（ラベルを色付きで表示）またはボタン列で代替
- `st.dataframe(on_select=...)` は Streamlit バージョン依存のため使わない

**② 詳細スコアカード**

選択工程の詳細を 3 カラムカードで表示:

```
┌──────────┐ ┌──────────┐ ┌──────────────┐
│  Cp      │ │  Cpk     │ │  総合判定     │
│  1.45    │ │  0.98    │ │  ✗ 不可      │
│▲分布幅十分│ │▼中心ずれ  │ │  Cpk < 1.00  │
└──────────┘ └──────────┘ └──────────────┘
📋 改善アクション
・平均値を規格中心（+0.12mm ずれ）に調整
・ばらつきは許容範囲内 — センタリングを優先
```

カード背景色は判定に応じて変化（良好=#f0fdf4, 要改善=#fff7ed, 不可=#fef2f2）

**③ ヒストグラム（visualize.py 生成）**

- matplotlib で PNG 生成 → `st.image()` で表示
- 規格外ゾーン: 赤（alpha=0.3）の axvspan（< LSL, > USL）
- 規格内ゾーン: 緑（alpha=0.1）の axvspan
- 正規曲線オーバーレイ: `scipy.stats.norm.pdf` で描画
- USL/LSL 縦線: 赤 dashed
- 平均線（μ）: navy dashed
- タイトルに `Cp={cp:.2f}  Cpk={cpk:.2f}  n={n}` を表示
- フォント: `matplotlib.rcParams["font.family"] = "BIZ UDGothic"`

## 7. エラーハンドリング

| エラー状況 | 対処 |
|-----------|------|
| 測定値列に文字列混在 | 数値変換できない行を除外してカウント表示 |
| USL ≤ LSL | 入力時にリアルタイムバリデーション（st.error） |
| サンプル数 < 20 | 警告表示（計算は実行、信頼性に注意文を追加） |
| 全測定値が同一 | σ=0 → Cp/Cpk = ∞ を回避、エラーメッセージ表示 |
| 工程名列に欠損値 | 欠損行を除外 |
